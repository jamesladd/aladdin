Rule R1: Explicitly detect compose mode as a distinct state transition. When the add-in detects it is in compose mode (item exists but has no itemId), treat this as an explicit deselection event — re-read state from localStorage, notify the previously captured email if present, and clear it. Do not rely solely on ItemChanged or re-initialization to handle compose transitions.
Handle compose as a first-class path in both initialize() and ItemChanged handling, separate from the noItem and differentItem cases.

Rule R2: Before calling notify() with a previously captured email, always re-read state from localStorage to check if another taskpane instance has already notified and cleared that email. This prevents double-notify when multiple taskpane instances exist (e.g. Desktop compose opening in a separate window while the read-mode taskpane remains active).

Rule R3: When the add-in initializes, if there is a previously captured email in persisted state, and the current context is either compose mode (no itemId), has no item, or has a different item, treat this as a deselection of the previous email — call notify() with the saved email data and clear it before proceeding with any new item capture.

This rule covers all deselection scenarios:

User switches to a different read email (ItemChanged)
User starts composing (taskpane re-initializes in compose context)
User navigates away from any email (no item)
Taskpane re-activates in a different context

Rule R4: The /api/inboxinit POST (called via getCategories() inside _initCategories()) and the subsequent _ensureMasterCategories() call must only execute once every 8 hours. The timestamp of the last successful execution must be persisted in state under _lastCategoryInit and checked at the start of _initCategories(). If less than 8 hours have elapsed since the last execution, the entire _initCategories() method must return early without making any API call or modifying master categories. The 8-hour window must survive across taskpane activations via localStorage persistence.
